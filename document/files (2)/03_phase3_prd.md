**2.1 技术架构：Monorepo + 独立站点**

  -------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------
  **架构决策**   中文站和英文站采用独立前端应用（独立Next.js apps），共享同一个Supabase数据库和所有后端服务。新城市只需复制最相似的app目录，改配置文件，2周内上线。这是Baam最重要的技术决策。

  -------------- ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------

  -------------- --------------------------- ------------------------------- ---------------------------------------------------------------------
  **层级**       **技术**                    **职责**                        **多城市支持方式**

  **代码仓库**   Turborepo Monorepo          统一管理所有城市站点代码        apps/ny-chinese / apps/oc-english / apps/la-chinese 独立Next.js应用

  **共享组件**   packages/ui                 Button/Card/Modal等基础UI组件   所有城市站点共用，一次修复全部受益

  **共享逻辑**   packages/api-client         Supabase客户端+API调用函数      城市通过region_id区分，相同API服务所有城市

  **共享AI**     packages/ai                 Claude API封装+Prompt模板       摘要/翻译/推荐逻辑复用，城市特化只需修改Prompt参数

  **数据库**     单一Supabase实例            PostgreSQL+pgvector             region_id字段隔离所有城市数据，无需多租户

  **部署**       Vercel（每个app独立项目）   各城市站点独立域名              ny.baam.us / oc.baam.us / la.baam.us，各自CI/CD

  **CDN/图片**   Cloudflare R2+Workers       全球加速                        所有城市共享R2存储桶，按region前缀组织
  -------------- --------------------------- ------------------------------- ---------------------------------------------------------------------
