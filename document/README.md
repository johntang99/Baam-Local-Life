## 🗄️ Database Quick Reference

Key tables in `04_supabase_schema.sql`:

```
regions             → 城市/县/街区 地区层级
categories          → 商家/文章/论坛 分类体系
profiles            → 用户 + Local Voices 人物主表
follows             → 关注关系
articles            → 新闻 + 生活指南 双用表 (content_vertical字段区分)
content_sources     → 新闻来源管理 (RSS自动抓取配置)
businesses          → 商家主表
business_locations  → 商家地址 (支持多地址)
forum_threads       → 论坛帖子 (含AI字段: ai_summary/ai_intent/ai_merchant_ids)
forum_replies       → 帖子回复
voice_posts         → Local Voices 内容主表
follows             → Follow关系
leads               → 商家线索 (来自指南/商家页/Voices页)
ai_jobs             → AI任务异步队列
search_logs         → 搜索行为日志 (AI训练数据)
```

## 🚀 Getting Started (for Engineers)

### 1. Database Setup
```bash
# Copy 04_supabase_schema.sql content
# Go to Supabase Dashboard → SQL Editor → Paste & Run
# Run section by section (17 sections)
```

### 2. Environment Variables needed
```env
NEXT_PUBLIC_SUPABASE_URL=your_supabase_url
NEXT_PUBLIC_SUPABASE_ANON_KEY=your_anon_key
SUPABASE_SERVICE_ROLE_KEY=your_service_key
ANTHROPIC_API_KEY=your_claude_api_key
CLOUDFLARE_R2_ACCESS_KEY=your_r2_key
CLOUDFLARE_R2_SECRET_KEY=your_r2_secret
CLOUDFLARE_R2_BUCKET=baam-media
```

### 3. Tech Stack
```
Frontend:   Next.js 15 (App Router) + TypeScript + Tailwind CSS
Database:   Supabase (PostgreSQL + pgvector)
AI:         Claude Sonnet (complex) + Claude Haiku (batch)
Search:     Meilisearch (self-hosted)
Storage:    Cloudflare R2
Deploy:     Vercel + Supabase
Automation: n8n (news RSS + AI workflows)
Monitoring: Langfuse (AI cost tracking)
```
